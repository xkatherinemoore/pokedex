Game Progress Summary/Homepage (index.html)

  1. Show/Hide About Section:

    The About Section (#about) display is toggled using buttons (#close-about & #open-about) and jQuery's .show() and .hide() methods. The div objects that are toggled are accessed using jQuery selectors, and when clicked, they will initiate the .show() and .hide() methods for the relevant divs.

  2. Load Relevant Game Data:

    When the user selects one of the radio button options to choose a game version (change event), a picture is added to the #game-photo element to display the game selection. Each game version also contains different set of Pokemon to collect, and these totals are saved in relevant variables within the JavaScript file. They are displayed in the #summary-pokemon section.

  3. Add Badges:

    The user is able to add and remove gym badges by clicking within the div elements. The script to do this utilizes jQuery to select the .badge class elements, and when one is clicked, it fires an event. The event object is then used to determine which of the .badge elements was clicked (return the id of the target object). If the clicked element already contains an img element, then it is removed. If not, the correct image path is found based on the target, a new image element is created with this path as the src attribute, and it is inserted as the first child of the div.

  4. Update Current Pokedex Table:

    The user adds a Pokemon to the Current Pokedex table via a form (#add-pokemon). This form makes use of an HTML datalist to ensure that only valid entries are added. On submission, Javascript is used to retrieve the submitted value and lookup information in the pokedex array (pokedex.js). The object returned from the pokedex array contains information which is added into td elements, which are then added into a single tr element that is added to the tbody via JavaScript's .appendChild() method. Pokemon count variables are incremented (since the table indicates that a Pokemon was caught) and updated, and the last line of the submission function clears the form using the .reset() method.

  5. Delete Table Entry:

    To delete an entry on the Current Pokedex table (#current-pokedex), the user can click on the trash can button that was added in the right-most column of the table entry. Event delegation is used because the delete button does not exist when the page is initially loaded. When the user clicks within the #current-pokedex table, an event is fired. If the event.target has a class name of .delButton, then the code executes to delete the table row element. This code also decrements pokemon count variables and updates the page.
